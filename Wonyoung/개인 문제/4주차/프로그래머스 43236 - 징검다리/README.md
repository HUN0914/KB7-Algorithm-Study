### 문제 설명

출발지점부터 distance만큼 떨어진 곳에 도착지점이 있습니다. 그리고 그사이에는 바위들이 놓여있습니다. 바위 중 몇 개를 제거하려고 합니다.
예를 들어, 도착지점이 25만큼 떨어져 있고, 바위가 [2, 14, 11, 21, 17] 지점에 놓여있을 때 바위 2개를 제거하면 출발지점, 도착지점, 바위 간의 거리가 아래와 같습니다.

| 제거한 바위의 위치 | 각 바위 사이의 거리 | 거리의 최솟값 |
| ------------------ | ------------------- | ------------- |
| [21, 17]           | [2, 9, 3, 11]       | 2             |
| [2, 21]            | [11, 3, 3, 8]       | 3             |
| [2, 11]            | [14, 3, 4, 4]       | 3             |
| [11, 21]           | [2, 12, 3, 8]       | 2             |
| [2, 14]            | [11, 6, 4, 4]       | 4             |

위에서 구한 거리의 최솟값 중에 가장 큰 값은 4입니다.

출발지점부터 도착지점까지의 거리 distance, 바위들이 있는 위치를 담은 배열 rocks, 제거할 바위의 수 n이 매개변수로 주어질 때, 바위를 n개 제거한 뒤 각 지점 사이의 거리의 최솟값 중에 가장 큰 값을 return 하도록 solution 함수를 작성해주세요.

### 제한사항

- 도착지점까지의 거리 distance는 1 이상 1,000,000,000 이하입니다.
- 바위는 1개 이상 50,000개 이하가 있습니다.
- n 은 1 이상 `바위의 개수` 이하입니다.

### 입출력 예

| distance | rocks               | n    | return |
| -------- | ------------------- | ---- | ------ |
| 25       | [2, 14, 11, 21, 17] | 2    | 4      |



### 문제 풀이 방법

이 문제는 "어떤 바위를 지울까?"가 아니라 "최솟값을 `X`로 설정했을 때, 지워야 하는 바위가 `n`개 이하인가?"라는 결정 문제로 변환하여 풀이



#### 1. 이분 탐색 범위 설정

- **시작값 (`start`)**: 1 (바위 사이의 최소 거리)
- **종료값 (`end`)**: `distance` (바위 사이의 최대 가능 거리)



#### 2. 거리 계산 로직

- `rocks` 배열을 정렬한 뒤, 각 바위 사이의 간격을 계산하여 `distanceRock` 배열에 저장합니다.
  - `시작점 ~ 첫 번째 바위`
  - `바위 ~ 바위`
  - `마지막 바위 ~ 도착점`
- 설정한 기준값(`middle`)보다 누적 거리(`d`)가 작으면 해당 바위를 **제거**(`cnt++`)합니다.
- 기준값보다 크거나 같으면 바위를 유지하고 다음 누적을 위해 거리를 초기화합니다.



#### 3. 조건에 따른 범위 이동

- **제거한 바위 수(`cnt`) > n**: 기준값이 너무 커서 바위를 너무 많이 지웠다는 뜻입니다. `end`를 줄여 기준값을 낮춥니다.
- **제거한 바위 수(`cnt`) <= n**: 기준값을 만족하면서 바위를 $n$개 이하로 지울 수 있습니다. 현재 값을 저장하고, 더 큰 최솟값이 있는지 확인하기 위해 `start`를 키웁니다.
