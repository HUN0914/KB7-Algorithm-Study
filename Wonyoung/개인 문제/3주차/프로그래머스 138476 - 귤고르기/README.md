### 문제 설명

경화는 과수원에서 귤을 수확했습니다. 경화는 수확한 귤 중 'k'개를 골라 상자 하나에 담아 판매하려고 합니다. 그런데 수확한 귤의 크기가 일정하지 않아 보기에 좋지 않다고 생각한 경화는 귤을 크기별로 분류했을 때 서로 다른 종류의 수를 최소화하고 싶습니다.

예를 들어, 경화가 수확한 귤 8개의 크기가 [1, 3, 2, 5, 4, 5, 2, 3] 이라고 합시다. 경화가 귤 6개를 판매하고 싶다면, 크기가 1, 4인 귤을 제외한 여섯 개의 귤을 상자에 담으면, 귤의 크기의 종류가 2, 3, 5로 총 3가지가 되며 이때가 서로 다른 종류가 최소일 때입니다.

경화가 한 상자에 담으려는 귤의 개수 `k`와 귤의 크기를 담은 배열 `tangerine`이 매개변수로 주어집니다. 경화가 귤 k개를 고를 때 크기가 서로 다른 종류의 수의 최솟값을 return 하도록 solution 함수를 작성해주세요.

------

### 제한사항

- 1 ≤ `k` ≤ `tangerine`의 길이 ≤ 100,000
- 1 ≤ `tangerine`의 원소 ≤ 10,000,000

------

##### 입출력 예

| k    | tangerine                | result |
| ---- | ------------------------ | ------ |
| 6    | [1, 3, 2, 5, 4, 5, 2, 3] | 3      |
| 4    | [1, 3, 2, 5, 4, 5, 2, 3] | 2      |
| 2    | [1, 1, 1, 1, 2, 2, 2, 3] | 1      |



## 문제 풀이

#### : HashMap & 정렬

이 문제는 귤의 크기 그 자체보다 **'각 크기별로 귤이 몇 개 있는가'**가 훨씬 중요합니다.



#### 1. 자료구조 선택: `HashMap` vs `int[]`

- **`int[]` (기존 방식)**: 크기 1,000만인 배열 선언 시 약 40MB의 메모리가 고정 할당되며, 귤의 종류가 적을 경우 메모리 낭비가 심함 ($O(MAX)$).
- **`HashMap` (선택 방식)**: 실제 존재하는 귤의 종류만큼만 메모리를 사용하며, 귤의 크기가 1억 이상으로 커져도 안전하게 처리 가능 ($O(N)$).



#### 2. 알고리즘 단계

1. **빈도수 계산**: `HashMap`을 사용하여 `크기(Key)` : `개수(Value)` 형태로 저장합니다.
2. **값 추출 및 정렬**: Map의 `values()`만 뽑아 리스트에 담은 후, **내림차순**으로 정렬합니다. 개수가 많은 귤이 리스트의 앞쪽에 오게 됩니다.
3. **그리디 선택**: 리스트를 순회하며 귤의 개수를 누적 합산(`sum`)합니다.
4. **종료 조건**: 누적 합산이 $k$보다 크거나 같아지는 순간의 순회 횟수가 곧 최소 종류의 수(`answer`)가 됩니다.
