### 문제 설명

`M x N` 크기의 격자 모양 지도가 주어지며, 출발지 `(0,0)`에서 목적지 `(M-1, N-1)`까지 이동하는 최단 거리를 구하는 문제입니다.

- 이동 시 상하좌우로 1칸씩 움직일 수 있습니다.
- 지도에는 지나갈 수 없는 함정(`hole`)이 존재합니다.
- 딱 한 번, '신비로운 신발'을 사용하여 함정을 뛰어넘어 한 방향으로 2칸을 동시에 이동할 수 있습니다. (이때 소요 시간은 똑같이 +1입니다.)



### 풀이: BFS + 상태 공간 확장 (3차원 방문 처리)

- 이 문제는 백준의 **'벽 부수고 이동하기'**, '말이 되고픈 원숭이'와 완전히 동일한 메커니즘을 가집니다. 

- 단순 좌표 탐색을 넘어 '특수 능력 사용 여부'라는 상태를 큐와 방문 배열에 추가해야 합니다.



#### 1. 상태 공간의 확장 (3차원 배열)

단순 `visited[row][col]`로는 신발을 쓰고 방문한 경우와 쓰지 않고 방문한 경우를 구분할 수 없습니다. 따라서 3차원 배열을 선언합니다.

- `visited[r][c][0]`: 신발을 **사용하지 않고** `(r, c)`에 도달한 최단 거리
- `visited[r][c][1]`: 신발을 **이미 사용하고** `(r, c)`에 도달한 최단 거리



#### 2. 두 가지 이동 분기

큐에서 현재 상태를 꺼내어 상하좌우(`directions`)를 탐색할 때 두 가지 행동을 취합니다.

1. **일반 이동**: 신발 사용 여부(`used`)를 그대로 유지한 채 상하좌우로 1칸 이동합니다.
2. **신발 점프**: 아직 신발을 쓰지 않았다면(`used == 0`), 해당 방향으로 2칸(1칸 점프)을 건너뛰어 이동하며, 상태를 `used + 1`로 갱신합니다.
