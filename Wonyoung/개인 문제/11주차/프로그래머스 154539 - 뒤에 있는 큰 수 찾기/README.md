### 문제 설명
정수로 이루어진 배열 `numbers`가 주어집니다. `numbers`의 각 원소에 대해 자신보다 뒤에 있는 숫자 중에서 자신보다 크면서 가장 가까이 있는 숫자를 찾아야 합니다. 이러한 숫자를 "뒷 큰수"라고 부릅니다. 만약 뒷 큰수가 존재하지 않는 원소는 -1을 기록합니다. 모든 원소에 대한 뒷 큰수를 차례대로 담은 배열을 반환하는 것이 목표입니다.

**제한사항:**
*   `3 <= numbers의 길이 <= 1,000,000`
*   `1 <= numbers[i] <= 1,000,000`

**입력 형식:**
*   `numbers`: 뒷 큰수를 찾을 정수 배열.

**출력 형식:**
*   `numbers`의 각 원소에 대한 뒷 큰수를 차례대로 담은 정수 배열 `answer`를 반환합니다.

**입력 예시 1:**
```
numbers = [2, 3, 3, 5]
```
**출력 예시 1:**
```
[3, 5, 5, -1]
```
**설명 1:**
*   2의 뒷 큰수는 3입니다.
*   3의 뒷 큰수는 5입니다.
*   3의 뒷 큰수는 5입니다.
*   5는 뒤에 큰 수가 없습니다.

**입력 예시 2:**
```
numbers = [9, 1, 5, 3, 6, 2]
```
**출력 예시 2:**
```
[-1, 5, 6, 6, -1, -1]
```
**설명 2:**
*   9는 뒤에 큰 수가 없습니다.
*   1의 뒷 큰수는 5입니다.
*   5의 뒷 큰수는 6입니다.
*   3의 뒷 큰수는 6입니다.
*   6은 뒤에 큰 수가 없습니다.
*   2는 뒤에 큰 수가 없습니다.

### 💡 Wonyoung's 풀이 전략 & 트러블 슈팅
이 문제는 배열의 각 원소에 대해 "오른쪽에 있는 가장 가까운 큰 수"를 찾는 전형적인 문제로, **단조 스택(Monotonic Stack)** 또는 **단조 덱(Monotonic Deque)**을 활용하여 효율적으로 해결할 수 있습니다.

**풀이 전략:**
1.  **자료구조 선택**: `Deque` (자바에서는 `ArrayDeque` 구현체 사용)를 스택처럼 활용합니다. 이 `Deque`에는 아직 뒷 큰수를 찾지 못한 숫자들의 `[값, 인덱스]` 쌍을 저장합니다. Deque의 마지막 원소가 스택의 TOP이라고 생각합니다.
2.  **단조성 유지**: `Deque`에는 항상 `[값, 인덱스]` 쌍이 값이 감소하는 순서로 쌓이도록 유지합니다. (Top에서부터 아래로 갈수록 값이 커지는 형태)
3.  **순회 및 처리**:
    *   `numbers` 배열을 왼쪽에서 오른쪽으로 (`i = 0`부터 `n-1`까지) 순회합니다.
    *   현재 숫자 `numbers[i]`를 만났을 때, `Deque`의 마지막 원소(`dq.peekLast()`)가 `numbers[i]`보다 작은 경우를 확인합니다.
    *   만약 `dq.peekLast()[0]`이 `numbers[i]`보다 작다면, `numbers[i]`는 `dq.peekLast()`에 해당하는 숫자의 "뒷 큰수"가 됩니다. 따라서 `dq.pollLast()`로 해당 원소를 덱에서 제거하고, `answer` 배열의 해당 인덱스(`num[1]`)에 `numbers[i]`를 기록합니다. 이 과정을 `dq`가 비거나 `dq.peekLast()[0]`이 `numbers[i]`보다 크거나 같아질 때까지 반복합니다.
    *   위 `while` 루프가 끝난 후, 현재 숫자 `[numbers[i], i]`를 `dq.offerLast()`를 통해 덱의 마지막에 추가합니다.
4.  **남은 원소 처리**: `numbers` 배열의 모든 원소를 순회한 후, `Deque`에 남아있는 모든 원소들은 자신보다 뒤에 큰 수가 없다는 의미입니다. 이 원소들에 대해서는 `dq.pollLast()`로 꺼내서 `answer` 배열의 해당 인덱스에 `-1`을 기록합니다.
5.  **결과 반환**: 완성된 `answer` 배열을 반환합니다.

**Java 문법 및 자료구조 활용:**
*   `java.util.Deque` 인터페이스와 `java.util.ArrayDeque` 구현체를 사용하여 스택처럼 동작시킵니다. `offerLast()`는 push, `pollLast()`는 pop, `peekLast()`는 top과 유사하게 사용됩니다.
*   `int[]` 배열을 `Deque`에 저장하여 숫자 값과 원래 인덱스를 함께 관리합니다. 이는 뒷 큰수를 찾았을 때 `answer` 배열의 올바른 위치에 값을 갱신하기 위함입니다.

**시간 복잡도**: `O(N)`
*   배열 `numbers`를 한 번 순회합니다.
*   각 원소는 `Deque`에 최대 한 번 삽입되고, 최대 한 번 제거됩니다.
*   따라서 전체 연산 수는 `N`에 비례합니다.

**공간 복잡도**: `O(N)`
*   최악의 경우 (예: 배열이 내림차순으로 정렬된 경우), 모든 원소가 `Deque`에 저장될 수 있습니다.
*   `answer` 배열도 `N`개의 공간을 차지합니다.

**트러블 슈팅 및 주의할 점:**
*   **인덱스 관리**: 단순히 값만 저장하는 것이 아니라 `[값, 인덱스]` 쌍으로 저장해야 합니다. 뒷 큰수를 찾았을 때 `answer` 배열의 올바른 `index`에 값을 할당해야 하기 때문입니다.
*   **`Deque`의 비어있는 상태 확인**: `peekLast()`나 `pollLast()`를 호출하기 전에 `dq.isEmpty()`를 반드시 확인해야 `NoSuchElementException`을 방지할 수 있습니다.
*   **초기 `answer` 배열**: `answer` 배열은 기본적으로 0으로 초기화되지만, 나중에 -1로 설정되는 로직이 있으므로 초기값은 크게 중요하지 않습니다. 모든 원소에 대해 -1로 미리 초기화하거나, 마지막에 처리하는 방식으로 해결할 수 있습니다. 이 코드에서는 후자 방식을 사용합니다.
*   **오름차순/내림차순 배열**: 이러한 엣지 케이스들을 시뮬레이션해보면 단조 스택의 동작 원리를 더 명확히 이해할 수 있습니다.
    *   `[1, 2, 3, 4]` (오름차순): 덱에 `[1,0]` -> `[2,1]`이 들어올 때 `[1,0]` 처리 -> `[3,2]`가 들어올 때 `[2,1]` 처리 -> ...
    *   `[4, 3, 2, 1]` (내림차순): 덱에 모든 원소가 `[4,0], [3,1], [2,2], [1,3]` 순으로 쌓인 후, 마지막 `while` 루프에서 모두 -1로 처리됩니다.
