### 문제 설명
<p>정수 <code>n</code>을 입력받아 <code>n</code>의 약수를 모두 더한 값을 리턴하는 함수, <code>solution</code>을 완성해주세요.</p>

<h4>제한 사항</h4>
<ul>
<li><code>n</code>은 1 이상 3000 이하인 정수입니다.</li>
</ul>

<h4>입력 형식</h4>
<ul>
<li>하나의 정수 <code>n</code>이 주어집니다.</li>
</ul>

<h4>출력 형식</h4>
<ul>
<li><code>n</code>의 모든 약수의 합을 정수 형태로 반환합니다.</li>
</ul>

### 💡 Wonyoung's 풀이 전략 & 트러블 슈팅
이 문제는 주어진 자연수 `n`의 모든 약수(divisor)를 찾아 합을 반환하는 기본적인 알고리즘 문제입니다.

#### 풀이 전략
1.  **초기화**: 약수들의 합을 저장할 변수 `answer`를 0으로 초기화합니다.
2.  **반복**: 1부터 `n`까지의 모든 자연수 `i`에 대해 반복문을 실행합니다.
3.  **약수 판별**: 각 `i`가 `n`의 약수인지 확인하기 위해 `n`을 `i`로 나누었을 때 나머지가 0인지 (`n % i == 0`) 검사합니다.
4.  **합산**: `i`가 `n`의 약수라면, `answer`에 `i`를 더합니다.
5.  **반환**: 반복문이 끝난 후, `answer`에 저장된 최종 값을 반환합니다.

#### 자바 문법 활용
*   **`for` 루프**: 1부터 `n`까지 순차적으로 숫자를 증가시키며 약수를 탐색하는 데 사용되었습니다.
*   **`%` (나머지 연산자)**: 어떤 수가 다른 수의 약수인지 판별하는 핵심 연산자로 사용되었습니다. (예: `n % i == 0`은 `i`가 `n`의 약수임을 의미)
*   **`+=` (복합 대입 연산자)**: 약수를 찾을 때마다 `answer` 변수에 해당 약수를 누적하는 데 사용되었습니다.

#### 트러블 슈팅 및 고려사항
*   **시간 복잡도**: 이 풀이 방식은 1부터 `n`까지 모든 수를 확인하므로 시간 복잡도는 O(N)입니다. 문제의 제약 조건인 `n`이 3000 이하인 경우, 이 정도의 시간 복잡도로는 매우 빠르게 계산이 가능하여 성능 문제가 발생하지 않습니다.
*   **공간 복잡도**: `n`, `answer`, `i`와 같은 몇 개의 정수 변수만 사용하므로 공간 복잡도는 O(1)입니다.
*   **엣지 케이스**: `n`이 1인 경우, `for` 루프는 `i=1`일 때 한 번 실행되고 `1 % 1 == 0`이 참이 되어 `answer`에 1이 더해지므로 올바르게 1을 반환합니다.
*   **오버플로우**: `n`의 최대값이 3000이고, 3000의 약수의 합은 8784입니다. 이는 `int` 자료형이 표현할 수 있는 최대 값(약 20억)을 훨씬 넘지 않으므로 오버플로우를 걱정할 필요가 없습니다.
*   **최적화 (심화)**: 만약 `n`의 범위가 훨씬 커진다면(예: 10^9), `sqrt(n)`까지만 탐색하여 약수 `i`를 찾고 동시에 `n/i`도 약수임을 활용하는 최적화된 방법(`O(sqrt(N))`)을 고려할 수 있습니다. 하지만 현재 문제의 제약 조건에서는 불필요합니다.
