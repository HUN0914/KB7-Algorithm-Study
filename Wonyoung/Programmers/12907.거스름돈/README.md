## 문제 설명

Finn은 편의점에서 야간 아르바이트를 하고 있습니다. 야간에 손님이 너무 없어 심심한 Finn은 손님들께 거스름돈을 n 원을 줄 때 방법의 경우의 수를 구하기로 하였습니다.

예를 들어서 손님께 5원을 거슬러 줘야 하고 1원, 2원, 5원이 있다면 다음과 같이 4가지 방법으로 5원을 거슬러 줄 수 있습니다.

- 1원을 5개 사용해서 거슬러 준다.
- 1원을 3개 사용하고, 2원을 1개 사용해서 거슬러 준다.
- 1원을 1개 사용하고, 2원을 2개 사용해서 거슬러 준다.
- 5원을 1개 사용해서 거슬러 준다.

거슬러 줘야 하는 금액 n과 Finn이 현재 보유하고 있는 돈의 종류 money가 매개변수로 주어질 때, Finn이 n 원을 거슬러 줄 방법의 수를 return 하도록 solution 함수를 완성해 주세요.

##### 제한 사항

- n은 100,000 이하의 자연수입니다.
- 화폐 단위는 100종류 이하입니다.
- 모든 화폐는 무한하게 있다고 가정합니다.
- 정답이 커질 수 있으니, 1,000,000,007로 나눈 나머지를 return 해주세요

##### 입출력 예

| n    | money   | result |
| ---- | ------- | ------ |
| 5    | [1,2,5] | 4      |



## 💡 문제 풀이 핵심: 조합(Combination) vs 순열(Permutation)

이 문제는 거스름돈을 구성하는 **동전의 종류가 같으면 순서가 달라도 같은 경우**로 취급해야 합니다. (즉, 조합을 구하는 문제)

### 1. `for`문 순서의 비밀
DP 테이블을 갱신할 때 이중 반복문의 순서가 결과의 성격을 결정합니다.

* **동전 기준 루프 (현재 풀이)**
    ```java
    for (int coin : money) {
        for (int i = coin; i <= n; i++) {
            dp[i] += dp[i - coin];
        }
    }
    ```
    - 하나의 동전을 충분히 다 사용한 뒤 다음 동전으로 넘어가기 때문에, 동전의 순서가 오름차순으로 고정되는 효과가 있습니다.
    - 따라서 `1원 + 2원`과 `2원 + 1원`을 중복으로 계산하지 않고 **한 번만** 카운트합니다.

* **금액 기준 루프 (잘못된 접근)**
    ```java
    for (int i = 1; i <= n; i++) {
        for (int coin : money) {
            if (i >= coin) dp[i] += dp[i - coin];
        }
    }
    ```
    - 각 금액마다 모든 동전의 가능성을 열어두기 때문에, `1원+2원`과 `2원+1원`을 각각 다른 경우의 수로 모두 더하게 됩니다.

### 2. 점화식 설명
$$dp[i] = dp[i] + dp[i - \text{coin}]$$
- `dp[i]`: 현재 동전 종류들로 $i$원을 만드는 기존의 방법 수
- `dp[i - coin]`: 새로운 동전 `coin`을 반드시 하나 포함하여 $i$원을 만드는 방법 수
