### 매개 변수 탐색 (Parametric Search)

바위의 위치를 직접 조합(Combination)으로 고르고 제거하는 완전 탐색 기법은 바위 개수가 최대 50,000개이기 때문에 시간 초과로 불가능합니다.

따라서 역발상을 적용하여 "바위 사이의 최소 간격을 `middle`로 설정했을 때, 제거해야 하는 바위의 개수가 `n`개 이하인가?"라는 결정 문제로 바꾸어 **이분 탐색**으로 해결합니다.

#### 1. 시작점과 끝점 처리 (배열 확장)

- 바위 사이의 거리를 계산할 때 출발점(`0`)과 도착점(`distance`)이 반드시 필요합니다.
- `rocks` 배열을 정렬한 뒤, 앞뒤로 크기를 2 늘린 `newArr`을 생성하여 `[0, 바위들..., distance]` 형태로 안전하게 배치합니다.

#### 2. 이분 탐색 범위 설정

- 바위 사이의 최소 거리의 범위는 `start = 1`부터 `end = distance`까지입니다.

#### 3. 바위 제거 시뮬레이션 (`middle` 기준)

- 인접한 바위 사이의 거리를 누적 변수 `d`에 더해나갑니다.
- 만약 누적 거리 `d`가 우리가 설정한 기준인 `middle`보다 **작다면**, 그 바위를 **제거**해야 합니다. (`cnt++`)
- 누적 거리 `d`가 `middle`보다 **크거나 같다면**, 기준 조건을 만족하므로 해당 바위를 그대로 두고 누적 거리를 다시 `0`으로 초기화한 뒤 다음 바위로 넘어갑니다.

#### 4. 범위 좁히기

- 제거한 바위 개수 `cnt`가 허용치 `n`보다 **작거나 같다면**: `middle`이라는 최소 간격을 유지할 수 있다는 뜻이므로 정답을 백업하고, 더 큰 간격이 가능한지 확인하기 위해 우측 범위를 탐색합니다. (`start = middle + 1`)
- 제거한 바위 개수 `cnt`가 허용치 `n`보다 **크다면**: 너무 욕심내서 간격을 크게 잡았다는 뜻이므로 간격 기준을 낮추기 위해 좌측 범위를 탐색합니다. (`end = middle - 1`)
