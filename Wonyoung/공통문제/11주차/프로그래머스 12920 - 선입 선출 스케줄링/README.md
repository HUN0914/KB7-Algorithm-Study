### 매개 변수 탐색 (Parametric Search)

작업의 개수 `n`이 최대 **50,000,000**이기 때문에, 시간을 1초씩 늘려가며 시뮬레이션하면 무조건 시간 초과가 발생합니다. 따라서 "특정 시간 `time`까지 모든 코어가 처리할 수 있는 총 작업량이 `n` 이상이 되는가?"라는 최적화 문제를 결정 문제로 바꾸어 **이분 탐색**으로 해결합니다.

#### 1. 예외 처리 (초기 배정)

- 처음에 모든 코어(개수: `m`)는 비어있으므로 0초에 코어 개수만큼 작업을 동시에 받습니다.
- 만약 `n <= cores.length`라면 계산할 필요 없이 `n`번째 코어가 정답이므로 바로 `n`을 리턴합니다.

#### 2. 이분 탐색 범위 설정 (`start`, `end`)

- `start`: 최소 시간인 `1`초
- `end`: 가장 느린 코어가 모든 작업을 혼자 처리할 때 걸리는 최대 시간인 $\frac{\text{max(cores)} \times n}{m}$ 으로 상한선을 안전하게 설정합니다.

#### 3. 특정 `time` 동안 처리 가능한 작업량 계산

- 코어별로 `time` 동안 처리할 수 있는 작업량은 다음과 같습니다.
  - 딱 나누어떨어지면: $\frac{\text{time}}{\text{core}}$
  - 안 나누어떨어지면 (작업 중): $\frac{\text{time}}{\text{core}} + 1$
- 이 총합(`sum`)이 `n`을 넘어서는 최소 시간(`time`)을 이분 탐색으로 찾아냅니다.

#### 4. 직전 시간 기준 잔여 작업 처리

- 구한 최소 시간의 직전 시간인 `recent = time - 1`까지 처리된 총 작업량을 구하여 `n`에서 차감합니다.
- 이제 `time` 시점에 새로 작업을 받을 수 있는 코어(즉, `recent % core == 0`으로 직전에 작업을 딱 끝낸 코어)들을 순서대로 순회하며 `n`을 줄여나가고, `n == 0`이 되는 순간의 코어 번호를 반환합니다.
