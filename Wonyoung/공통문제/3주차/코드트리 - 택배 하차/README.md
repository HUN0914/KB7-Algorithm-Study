
# 📦 코드트리: 택배 하차 

## 1. 알고리즘 흐름
1. **[투입]** 지정된 열($c$)에서 중력으로 떨어뜨려 적재.
2. **[반복]** 격자 위에 택배가 없을 때까지 아래 사이클을 반복:
    - **좌측 하차:** 왼쪽으로 끝까지 뺄 수 있는 택배 중 **번호가 가장 작은 것** 제거.
    - **중력 작용:** 빈 공간으로 택배들 하강.
    - **우측 하차:** 오른쪽으로 끝까지 뺄 수 있는 택배 중 **번호가 가장 작은 것** 제거.
    - **중력 작용:** 빈 공간으로 택배들 하강.

---

## 2. 주요 로직 상세

### ① 택배 적재 (Stacking)
- `Map[c ~ c+w-1]` 범위를 확인해 가장 높은 층(`max_h`)을 찾음.
- 해당 위치 바로 위(`max_h + 1`)에 택배 배치.

### ② 하차 가능 체크 (Collision Check)
- **좌측 하차:** `grid[row_range][0 ~ 현재_좌측_열-1]`에 다른 택배 번호가 하나도 없어야 함.
- **우측 하차:** `grid[row_range][현재_우측_열+1 ~ N-1]`에 다른 택배 번호가 하나도 없어야 함.

### ③ 중력/재배치 (Relocation)
- **핵심:** 반드시 **아래쪽 행(Bottom-up)**에 있는 택배부터 검사.
- 각 택배 아래에 빈 공간(0)이 있다면, 장애물을 만날 때까지 한 칸씩 또는 한꺼번에 이동.

---

## 3. 구현 주의사항
* **순서 엄수:** `좌측 하차 -> 중력 -> 우측 하차 -> 중력` 순서를 정확히 지켜야 예제와 같은 결과가 나옴.
* **격자 업데이트:** 택배가 이동하거나 하차할 때마다 `int[][] grid` 정보를 즉시 갱신해야 충돌 판정에 오류가 없음.
* **인덱스:** 문제의 $c$(열)가 1부터 시작하는지 확인하고 0-based로 변환 필요.
