# [level 3] 표현 가능한 이진트리 - 150367 

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/150367) 

### 성능 요약
*   **메모리**: 약 10-20 MB (입력 배열 `numbers`의 길이 `N`이 최대 100,000일 때, `answer` 배열이 약 0.4MB를 차지하며, 이외의 문자열 및 재귀 스택 공간은 미미합니다.)
*   **시간**: 약 10-100 ms (각 `long` 숫자에 대해 이진 문자열 변환 및 재귀 검증이 `log(max_value)` 시간 복잡도를 가지므로, 총 `N * log(max_value)` 연산이 수행됩니다. `max_value`는 `2^63 - 1`이므로 `log(max_value)`는 약 63에 해당하며, `N`이 최대 100,000일 경우 총 630만 연산으로 매우 빠르게 처리됩니다.)

### 구분
카카오 2023 KAKAO BLIND RECRUITMENT (코딩테스트 연습)

### 문제 설명
주어진 `long` 타입 정수 배열 `numbers`에 대해 각 숫자가 "표현 가능한 이진트리"인지 판단하고, 그 결과를 `int` 배열로 반환하는 문제입니다. "표현 가능한 이진트리"는 다음 조건을 만족하는 이진트리를 의미합니다:

1.  **이진수 변환 및 포화 이진트리 형태로 변환**:
    *   각 숫자를 이진수로 변환합니다.
    *   변환된 이진수 문자열의 길이를 `L`이라고 할 때, `L`보다 크거나 같은 가장 작은 `2^k - 1` 형태의 길이(`target_L`)를 찾습니다. 이는 포화 이진트리(모든 노드가 0개 또는 2개의 자식을 가지며, 모든 리프 노드가 같은 깊이에 있는 트리)의 노드 개수에 해당합니다.
    *   원본 이진수 문자열의 앞에 `target_L - L`개의 '0'을 패딩하여 총 길이가 `target_L`인 문자열을 만듭니다. 이렇게 패딩된 문자열은 포화 이진트리의 노드를 중위 순회(inorder traversal)한 결과로 간주됩니다.

2.  **유효성 검사 규칙**:
    *   포화 이진트리 형태로 변환된 이진 문자열에 대해, 특정 규칙에 따라 유효성을 검사합니다.
    *   **핵심 규칙**: 어떤 노드의 값이 '0'인 경우, 해당 노드의 자식 노드(왼쪽 자식과 오른쪽 자식)는 반드시 모두 '0'이어야 합니다. 즉, '0' 노드는 '1' 자식을 가질 수 없습니다.
    *   어떤 노드의 값이 '1'인 경우, 해당 노드의 자식 노드는 '0' 또는 '1'이 될 수 있습니다.

**입력 형식:**
*   `numbers`: `long` 타입 정수들의 배열.
*   `numbers[i]`는 `1` 이상 `10^15` 이하이며, `long` 타입의 최댓값(`2^63 - 1`)까지 가능합니다.
*   `numbers` 배열의 길이는 1 이상 100,000 이하입니다.

**출력 형식:**
*   `int[]`: `numbers` 배열의 각 숫자에 대해 "표현 가능한 이진트리"이면 `1`, 아니면 `0`을 담은 배열을 반환합니다.

---
### 💡 풀이 방법
이 코드는 주어진 `long` 정수가 특정 규칙을 만족하는 "표현 가능한 이진트리"인지 판별하는 재귀적 접근 방식을 사용합니다.

1.  **이진수 변환 및 패딩**:
    *   `solution` 메서드는 `numbers` 배열의 각 `long` 숫자에 대해 다음 과정을 반복합니다.
    *   `Long.toBinaryString(numbers[i])`를 사용하여 해당 숫자를 이진수 문자열 `binStr`로 변환합니다.
    *   `binStr`의 현재 길이 `len`을 기준으로 `addZero()` 헬퍼 함수를 호출하여 필요한 '0' 패딩 개수를 계산합니다.
        *   `addZero(len)` 함수는 `len`보다 크거나 같은 최소의 `2^k` 값을 찾은 후 (예: `len=5`면 `2^k=8`), `(2^k - 1) - len` 만큼의 '0'을 추가해야 함을 계산하여 반환합니다. 이 `2^k - 1` 길이는 `binStr`을 포함할 수 있는 가장 작은 포화 이진트리의 총 노드 개수에 해당합니다.
    *   `result = (len & (len + 1)) == 0 ? binStr : "0".repeat(addZero(len)) + binStr;`
        *   이 조건문은 `len`이 이미 `2^k - 1` 형태(예: 1, 3, 7, 15...)인지 효율적으로 확인하는 비트 연산(`len & (len + 1) == 0`)을 사용합니다. `len`이 `2^k - 1` 형태이면 `len + 1`은 `2^k`가 되며, 이 둘의 AND 연산 결과는 `0`입니다. 이 경우 추가 패딩 없이 `binStr` 자체가 `result`가 됩니다.
        *   그렇지 않다면, `String.repeat(addZero(len))`을 이용하여 계산된 개수만큼의 '0'을 `binStr` 앞에 붙여 `result` 문자열을 완성합니다. 이렇게 `result`는 포화 이진트리의 노드 정보를 담은 문자열이 됩니다.

2.  **재귀적 유효성 검사 (`isBinaryTree` 함수)**:
    *   `isBinaryTree(result, length / 2 + 1, length / 2)` 함수를 호출하여 `result` 문자열이 "표현 가능한 이진트리" 규칙을 만족하는지 검사합니다.
        *   `length`는 패딩된 `result` 문자열의 총 길이입니다.
        *   `length / 2`는 현재 서브트리의 루트 노드 인덱스(`loc`)입니다. (예: 길이 7인 문자열 "0001010"의 루트는 인덱스 3인 '1'입니다.)
        *   `length / 2 + 1`은 `size` 매개변수로, 현재 서브트리의 크기 또는 다음 재귀 호출 시 자식 노드의 상대적 위치(`offset`)를 계산하는 데 사용됩니다.
    *   **종료 조건**: `size`가 1이 되면 (현재 노드가 리프 노드이면) 더 이상 자식 노드가 없으므로 유효하다고 판단하여 `true`를 반환합니다.
    *   **재귀 호출 로직**:
        *   `newSize = size / 2`를 계산하여 다음 단계의 자식 노드까지의 `offset`을 정합니다.
        *   `leftIdx = loc - newSize`, `rightIdx = loc + newSize`를 통해 현재 루트 노드의 왼쪽 및 오른쪽 자식 노드의 인덱스를 계산합니다.
        *   **핵심 규칙 적용**: `if (result.charAt(loc) == '0')` 조건문을 통해 현재 루트 노드가 '0'인지 확인합니다.
            *   만약 루트 노드가 '0'인데, 왼쪽 자식(`result.charAt(leftIdx)`)이나 오른쪽 자식(`result.charAt(rightIdx)`) 중 하나라도 '1'이면 규칙 위반이므로 즉시 `false`를 반환합니다.
        *   현재 노드가 규칙을 위배하지 않으면, `leftValid = isBinaryTree(result, newSize, leftIdx)`와 `rightValid = isBinaryTree(result, newSize, rightIdx)`를 호출하여 왼쪽 및 오른쪽 서브트리에 대해 재귀적으로 검사를 진행합니다.
        *   최종적으로 `leftValid && rightValid`를 반환하여 두 서브트리가 모두 유효해야만 `true`를 반환합니다.

**Java 문법 활용 및 주의사항:**
*   **`Long.toBinaryString()`**: `long` 타입의 숫자를 이진수 형태의 `String`으로 변환하는 데 사용됩니다.
*   **`String.repeat(int)`**: JDK 11 이상에서 도입된 메서드로, 특정 문자열을 지정된 횟수만큼 반복하여 새로운 문자열을 생성합니다. (버전 호환성 주의)
*   **비트 연산 `(len & (len + 1)) == 0`**: `len`이 `2^k - 1` 형태의 숫자인지 효율적으로 확인하는 데 사용됩니다. 이는 `len`과 `len + 1`의 이진 표현에서 공통으로 `1`로 설정된 비트가 없다는 것을 의미합니다.
*   **재귀 함수**: `isBinaryTree` 함수는 포화 이진트리 구조를 재귀적으로 탐색합니다. `long` 값의 이진 표현 최대 길이가 약 63이므로, 재귀 깊이가 최대 6~7단계에 불과하여 스택 오버플로우 발생 가능성은 거의 없습니다.
*   **인덱스 계산**: 이진 문자열이 포화 이진트리의 중위 순회 결과처럼 나열된다는 특징을 이용하여, `loc` (현재 노드 인덱스), `newSize` (자식 노드까지의 상대적 거리)를 통해 자식 노드의 인덱스를 효율적으로 계산합니다. 이러한 인덱스 계산의 정확성이 문제 해결의 핵심입니다.
