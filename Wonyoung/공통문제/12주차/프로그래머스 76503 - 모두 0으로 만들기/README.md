# [level 3] 모두 0으로 만들기 - 76503 

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/76503) 

### 성능 요약
메모리: 30 - 60 MB, 시간: 5 - 50 ms (N <= 300,000 기준)
*   시간 복잡도: O(N) - 트리의 모든 노드와 간선을 한 번씩 방문하므로 선형 시간에 해결됩니다.
*   공간 복잡도: O(N) - 인접 리스트, 값 배열, 방문 배열, 부모 배열, DFS 스택 등에 N에 비례하는 공간을 사용합니다.

### 구분
코딩테스트 연습

### 문제 설명
<p>트리로 주어진 모든 노드의 가중치를 0으로 만드는 최소 비용을 계산하는 문제입니다.</p>

<h4>문제 핵심 요구사항</h4>
<ul>
    <li>주어진 그래프는 항상 <strong>트리 형태</strong>입니다.</li>
    <li>각 노드에는 <code>a[i]</code>에 해당하는 가중치가 있습니다.</li>
    <li>작업: 인접한 두 노드 <code>u</code>와 <code>v</code> 사이에서 원하는 만큼의 가중치를 이동시킬 수 있습니다.</li>
    <li>비용: 가중치 <code>x</code>를 이동시키면 비용은 <code>|x|</code>만큼 발생하며, 이 비용은 누적됩니다.</li>
    <li>목표: 모든 노드의 가중치를 0으로 만드는 최소 비용을 구해야 합니다.</li>
    <li>특수 조건: 만약 모든 노드의 가중치 합이 0이 아니라면, 모든 노드를 0으로 만드는 것이 불가능하므로 -1을 반환해야 합니다.</li>
</ul>

<h4>입력 형식</h4>
<ul>
    <li><code>a</code>: 각 노드의 가중치를 나타내는 <code>int</code> 배열 (<code>a.length</code>는 노드의 개수 <code>n</code>).</li>
    <li><code>edges</code>: 그래프의 간선 정보를 나타내는 <code>int[][]</code> 배열. 각 원소 <code>[u, v]</code>는 <code>u</code>번 노드와 <code>v</code>번 노드가 연결되어 있음을 의미합니다.</li>
</ul>

<h4>출력 형식</h4>
<ul>
    <li>모든 노드의 가중치를 0으로 만드는 최소 비용을 <code>long</code> 타입으로 반환합니다.</li>
    <li>만약 불가능하다면 -1을 반환합니다.</li>
</ul>

---
### 💡 풀이 방법
이 문제는 트리의 특성과 가중치 이동의 특성을 활용하여 해결합니다. 핵심 아이디어는 리프 노드부터 루트 노드 방향으로 값을 정산하여 올라가는 것입니다.

1.  **초기 합 검사**:
    *   가장 먼저, 모든 노드의 가중치 합을 계산합니다. 만약 이 합이 0이 아니면, 어떤 방식으로 가중치를 이동하더라도 전체 합은 변하지 않으므로 모든 노드를 0으로 만들 수 없습니다. 이 경우 즉시 -1을 반환합니다.

2.  **그래프 구성**:
    *   주어진 `edges` 정보를 바탕으로 인접 리스트(`HashMap<Integer, List<Integer>>`)를 사용하여 트리를 구현합니다. 이는 양방향 간선을 추가하여 무방향 그래프로 만듭니다.

3.  **반복적 DFS (Iterative DFS)를 이용한 순서 결정**:
    *   재귀 DFS는 트리의 깊이가 매우 깊을 경우 `StackOverflowError`를 발생시킬 수 있으므로, 스택(`Deque`)을 활용한 반복적 DFS를 사용합니다.
    *   DFS를 수행하여 `order` 리스트에 노드들을 저장합니다. 이 `order` 리스트는 스택에서 노드를 `pollLast()` (꺼낼) 때마다 추가되므로, 리프 노드들이 먼저 나오고 그 부모 노드들이 나중에 나오는 "역위상 정렬" 또는 "후위 순회(Post-order traversal)의 역순"과 같은 순서가 됩니다.
    *   `parent` 배열을 사용하여 각 노드의 부모를 기록합니다. 이는 이후 가중치를 부모 노드로 전달하는 데 사용됩니다.

4.  **가중치 정산 및 비용 계산**:
    *   `order` 리스트를 <strong>역순으로 순회</strong>합니다 (즉, 리프 노드부터 시작하여 루트 노드 방향으로).
    *   각 노드 `cur`에 대해 다음을 수행합니다:
        *   `answer`에 `Math.abs(values[cur])`을 더합니다. 이는 현재 노드 `cur`의 가중치를 0으로 만들기 위해 이동시켜야 하는 양의 절대값을 누적하는 것입니다.
        *   만약 `cur` 노드가 루트 노드가 아니라면 (`parent[cur] != -1`): `cur` 노드의 현재 가중치(`values[cur]`)를 부모 노드(`parent[cur]`)로 이동시킵니다. 즉, `values[parent[cur]] += values[cur];`를 수행합니다. 이로써 `cur` 노드의 가중치는 (개념적으로) 부모에게 전가되어 0이 되고, 부모 노드의 가중치는 `cur` 노드의 가중치만큼 변하게 됩니다.
    *   이 과정을 리프 노드부터 시작하여 루트 노드까지 반복하면, 최종적으로 루트 노드를 제외한 모든 노드의 가중치는 0이 되고, 루트 노드의 가중치는 초기 전체 가중치 합계가 됩니다. 초기 합이 0이었으므로 최종적으로 루트 노드의 가중치 또한 0이 됩니다.
    *   이렇게 누적된 `answer`가 모든 노드를 0으로 만드는 최소 비용이 됩니다.

5.  **자료형 주의**:
    *   노드의 가중치 `a[i]`는 최대 10^8이고 노드 수는 3 * 10^5까지 가능하므로, 가중치들의 합이나 이동 비용이 `int` 자료형의 범위를 넘어설 수 있습니다. 따라서 `values` 배열과 `answer` 변수는 `long` 타입으로 선언하고 사용해야 합니다. (`int[] a`는 입력만 받고, 실제 연산은 `long[] values`로 합니다.)
