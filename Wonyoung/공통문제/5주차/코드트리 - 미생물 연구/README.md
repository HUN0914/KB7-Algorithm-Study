# 미생물 연구

평균 180분/ 41% 정답률/ 총 제출 4,260회

N×N 크기의 격자에서 진행되는 미생물 투입 및 이동 시뮬레이션입니다.



#### 1. 미생물 투입

- **(r1, c1)** ~ **(r2, c2)** 영역에 새로운 미생물 무리를 투입합니다.
- 영역 내 기존 미생물은 모두 사라지고 새 미생물이 자리를 차지합니다.
- **중요:** 기존 무리가 새 무리에 의해 공격받아 **영역이 둘 이상으로 분리**될 경우, 해당 무리는 용기에서 완전히 소멸합니다.

#### 2. 배양 용기 이동 (재배치)

매 실험마다 모든 미생물을 다음 우선순위에 따라 새 용기로 옮깁니다.

1. **선택 순위:** 영역이 넓은 무리 우선 → 같다면 먼저 투입된 무리 우선.
2. **배치 규칙:** 기존 형태 유지, 범위 이탈 및 겹침 금지.
3. **배치 위치:** **최소 x좌표** 우선 → 같다면 **최소 y좌표** 위치에 배치.

- 이동이 불가능한 무리는 그대로 소멸합니다.

#### 3. 실험 결과 기록

인접한(상하좌우 맞닿은) 미생물 무리 쌍의 성과를 합산합니다.

- **계산식:** $\sum (\text{무리 A의 넓이} \times \text{무리 B의 넓이})$
- 맞닿은 면이 여러 곳이라도 한 쌍으로 간주하여 한 번만 계산합니다.



#### *N*과 Sam이 실험에서 투입한 미생물의 정보가 주어질 때, 각 실험의 결과를 출력하는 프로그램을 작성하세요.

##### 📥 입력

```
8 4
2 2 5 6
2 3 5 8
2 0 5 3
1 1 6 6
```

##### 📤 출력

```
0
165
108
325
```



### 풀이 방법

#### 1. 미생물 투입 및 소멸 (`injectGerm`)

- **영역 점유**: 주어진 좌표 범위 $(r1, c1)$ ~ $(r2, c2)$를 순회하며 새로운 `num`으로 채웁니다.
- **분리 확인 (`checkDivided`)**: 기존의 미생물들($1 \dots num-1$)을 전수 조사합니다.
  - **BFS 활용**: 각 미생물 번호에 대해 연결 요소(Connected Component)의 개수를 구합니다.
  - 덩어리 개수가 2개 이상이 되면 `removeGerm`을 통해 격자에서 제거합니다.



#### 2. 우선순위에 따른 재배치 (`moveBoard`)

모든 미생물을 새 용기(`newBoard`)로 옮기는 과정입니다.

- **우선순위 결정**: 각 미생물의 현재 크기를 측정하여 [크기 내림차순, ID 오름차순]으로 `PriorityQueue`에 담습니다.
- **최적 위치 탐색**:
  - 각 무리의 모든 좌표 정보를 `germPos`에 저장합니다.
  - $N \times N$ 전체를 순회하며 `isMovable` 함수로 해당 위치에 배치 가능한지 확인합니다.
  - **배치 기준**: $x$좌표(코드상 열)가 작은 순서, 그 다음 $y$좌표(코드상 행)가 작은 순서대로 가장 먼저 발견되는 곳에 고정합니다.
- **충돌 검사 (`isMovable`)**: 형태를 유지한 채 이동했을 때, 격자 범위를 벗어나거나 이미 다른 미생물이 배치된(`newBoard[nr][nc] != 0`) 곳이 있는지 체크합니다.



#### 3. 실험 결과 점수 계산 (`recordResult`)

- **크기 측정**: 현재 생존한 미생물들의 영역 넓이를 배열에 저장합니다.
- **인접 쌍 찾기**: 격자를 순회하며 상하좌우에 다른 ID의 미생물이 있는지 확인합니다.
- **중복 제거**: `boolean[][] neighbor` 배열을 사용하여 이미 계산한 무리 쌍 $(A, B)$은 다시 계산하지 않도록 관리합니다.
- **최종 점수**: 인접한 것으로 판명된 모든 쌍에 대해 `size[A] * size[B]`를 누적 합산합니다.
