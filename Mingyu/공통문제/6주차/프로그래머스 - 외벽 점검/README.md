## 📝 문제 설명

원형 외벽에 여러 개의 취약 지점이 있다.

각 친구는 1시간 동안 이동할 수 있는 거리가 정해져 있으며, 친구를 최소한으로 투입해서 모든 취약 지점을 점검해야 한다.

친구는 시계 방향 또는 반시계 방향으로 이동할 수 있다.

단, 모든 친구를 투입해도 모든 취약 지점을 점검할 수 없다면 `-1`을 반환한다.

---

## 📥 입력 형식

프로그래머스 함수형 문제이므로 별도의 표준 입력은 없다.

```java
int solution(int n, int[] weak, int[] dist)
```

| 매개변수 | 설명                               |
| -------- | ---------------------------------- |
| `n`      | 원형 외벽의 총 길이                |
| `weak`   | 취약 지점의 위치 배열              |
| `dist`   | 각 친구가 이동할 수 있는 거리 배열 |

---

## 📤 출력 형식

모든 취약 지점을 점검하기 위해 필요한 최소 친구 수를 반환한다.

```java
return minCnt;
```

불가능한 경우:

```java
return -1;
```

---

## 🔒 제한 사항

| 항목             | 제한                     |
| ---------------- | ------------------------ |
| `n`              | `1 <= n <= 200`          |
| `weak.length`    | `1 <= weak.length <= 15` |
| `weak[i]`        | `0 <= weak[i] <= n - 1`  |
| `dist.length`    | `1 <= dist.length <= 8`  |
| `dist[i]`        | `1 <= dist[i] <= 100`    |
| `weak` 정렬 여부 | 오름차순 정렬            |

---

## 🧪 입출력 예시

|   n | weak               | dist           | result |
| --: | ------------------ | -------------- | -----: |
|  12 | `[1, 5, 6, 10]`    | `[1, 2, 3, 4]` |      2 |
|  12 | `[1, 3, 4, 9, 10]` | `[3, 5, 7]`    |      1 |

## 💡 핵심 아이디어

이 문제의 핵심은 **원형 외벽을 직선 배열처럼 바꾸는 것**이다.

원형을 직접 `% n`으로 처리하면 구현이 복잡해진다.

따라서 `weak` 배열을 2배로 늘린다.

```text
n = 12
weak = [1, 5, 6, 10]

extendWeak = [1, 5, 6, 10, 13, 17, 18, 22]
```

즉, 뒤쪽에는 기존 취약 지점에 `n`을 더한 값을 붙인다.

```java
extendWeak[i] = weak[i];
extendWeak[i + weak.length] = weak[i] + n;
```

이렇게 하면 원형에서 한 바퀴 도는 상황을 직선 위에서 처리할 수 있다.

---

## 🔁 왜 배열을 2배로 늘리는가?

예를 들어 외벽 길이가 `12`이고 취약 지점이 다음과 같다고 하자.

```text
weak = [1, 5, 6, 10]
```

원형에서 `10`부터 시작하면 다음 취약 지점은 `1`이다.

하지만 직선 배열에서는 `1`을 한 바퀴 뒤의 위치인 `13`으로 볼 수 있다.

```text
10 -> 13
```

그래서 시작점이 `10`일 때 검사 대상은 다음처럼 표현된다.

```text
[10, 13, 17, 18]
```

이제 원형 처리를 하지 않고도 단순한 직선 탐색이 가능해진다.

---

## 🧑‍🤝‍🧑 왜 dist 순열을 봐야 하는가?

친구들의 이동 거리가 다르기 때문에 **투입 순서에 따라 결과가 달라진다.**

예를 들어:

```text
dist = [1, 2, 3, 4]
```

다음 두 순서는 서로 다른 결과를 만들 수 있다.

```text
[1, 2, 3, 4]
[4, 2, 1, 3]
```

따라서 `dist`의 모든 순열을 만들어 확인해야 한다.

최대 친구 수는 8명이므로:

$$
8! = 40320
$$

`weak.length`도 최대 15이므로 전체 탐색이 가능하다.

---

## 🧮 시간복잡도 계산

`w = weak.length`, `d = dist.length`라고 하면:

$$
O(d! \times w^2)
$$

최대값은 다음과 같다.

```text
8! * 15 * 15
= 40320 * 225
= 9,072,000
```

약 900만 번 정도의 검사이므로 Java에서도 충분히 가능하다.

---

## 🧭 풀이 흐름

```text
1. weak 배열을 2배로 늘려 extendWeak를 만든다.

2. dist 배열의 모든 순열을 만든다.

3. 각 순열마다 weak의 모든 지점을 시작점으로 잡아본다.

4. 현재 친구가 커버할 수 있는 가장 먼 위치 cover를 계산한다.

5. 다음 취약 지점이 cover 안에 있으면 그대로 통과한다.

6. cover를 벗어난 취약 지점이 나오면 다음 친구를 투입한다.

7. 친구 수가 dist.length를 넘으면 실패한다.

8. 모든 취약 지점을 점검했다면 최소 친구 수를 갱신한다.
```

---

## ⚠️ 주의할 점

### 1. 원형을 직접 `% n`으로 처리하지 말기

가능은 하지만 시작점, 방향, 방문 범위 관리가 복잡해진다.

이 문제는 `weak`를 2배로 확장하는 방식이 훨씬 단순하다.

---

### 2. 시작점은 모든 weak 지점을 확인해야 한다

원형에서는 어디서 출발하느냐에 따라 필요한 친구 수가 달라질 수 있다.

```java
for (int start = 0; start < wLen; start++) {
    ...
}
```

---

### 3. dist 순열이 필요하다

이동 거리가 큰 친구를 먼저 쓰는 것이 항상 최적이라고 단정할 수 없다.

따라서 모든 친구 투입 순서를 확인한다.

---

### 4. 친구 수 초과 여부를 반드시 확인한다

```java
if (cnt > dist.length) {
    break;
}
```

이 조건이 없으면 `order[cnt - 1]` 접근에서 배열 범위를 벗어날 수 있다.

---

### 5. 실패한 경우는 정답 갱신하지 않는다

```java
if (cnt <= dist.length) {
    minCnt = Math.min(minCnt, cnt);
}
```

친구를 모두 써도 점검하지 못한 경우는 제외해야 한다.

---

## ✅ 복잡도

| 항목       | 복잡도             |
| ---------- | ------------------ |
| 시간복잡도 | $O(d! \times w^2)$ |
| 공간복잡도 | $O(w + d)$         |

단, `w = weak.length`, `d = dist.length`이다.
