## 📝 문제 설명

가로 길이 $n$, 세로 길이 $m$인 보물 지도가 주어진다.

시작 위치는 $(1, \ 1)$이고, 목적지는 $(n, \ m)$이다.

지도에는 함정이 존재하며, 함정 칸으로는 일반 이동할 수 없다.

이동 방법은 두 가지다.

| 이동 방법     | 설명                           | 소요 시간 |
| ------------- | ------------------------------ | --------- |
| 걷기          | 상하좌우 인접한 한 칸으로 이동 | $1$       |
| 신비로운 신발 | 상하좌우 방향으로 두 칸 점프   | $1$       |

신비로운 신발은 **한 번만 사용할 수 있다.**

신발을 사용하면 중간에 있는 함정은 뛰어넘을 수 있다.

단, 최종 도착 칸이 함정이면 이동할 수 없다.

목적지까지 갈 수 있는 최소 시간을 구하고, 갈 수 없다면 `-1`을 반환한다.

---

## 📥 입력

```text
n: 지도의 가로 길이
m: 지도의 세로 길이
hole: 함정 좌표 배열
```

좌표는 $(1, \ 1)$부터 시작한다.

```text
hole[i] = [a, b]

(a, b) 위치에 함정이 존재한다.
```

---

## 📤 출력

```text
(1, 1)에서 (n, m)까지 이동하는 최소 시간

이동할 수 없다면 -1
```

---

## ✅ 핵심 아이디어

이 문제는 일반적인 좌표 BFS처럼 `visited[x][y]`만 사용하면 안 된다.

같은 좌표에 도착했더라도 다음 두 상태는 서로 다르다.

```text
1. 신발을 아직 안 쓴 상태
2. 신발을 이미 쓴 상태
```

예를 들어 같은 $(x, \ y)$에 있어도 신발을 아직 안 썼다면 이후에 함정을 뛰어넘을 수 있다.

하지만 신발을 이미 썼다면 더 이상 점프할 수 없다.

따라서 BFS 상태를 다음처럼 잡는다.

```text
(x, y, used)

x, y : 현재 좌표
used : 신발 사용 여부
       0 = 아직 안 씀
       1 = 이미 씀
```

즉, 이 문제는 **좌표 + 신발 사용 여부**를 함께 관리하는 BFS 문제다.

---

## 🔍 방문 배열

```java
boolean[][][] visited = new boolean[n + 1][m + 1][2];
```

각 의미는 다음과 같다.

```text
visited[x][y][0]
= 신발을 안 쓴 상태로 (x, y)에 방문한 적이 있는가?

visited[x][y][1]
= 신발을 쓴 상태로 (x, y)에 방문한 적이 있는가?
```

`visited[x][y]`만 사용하면 틀릴 수 있다.

같은 칸이라도 신발을 사용했는지 여부에 따라 이후 이동 가능성이 달라지기 때문이다.

---

## 🚶 이동 방식

현재 상태에서 매 방향마다 두 가지 이동을 시도한다.

```text
1. 한 칸 걷기
2. 신발을 아직 안 썼다면 두 칸 점프
```

---

## 1️⃣ 한 칸 걷기

```text
현재 위치 → 인접한 한 칸
```

이동 조건은 다음과 같다.

```text
1. 지도 범위 안이어야 한다.
2. 도착 칸이 함정이면 안 된다.
3. 같은 신발 사용 상태로 방문한 적이 없어야 한다.
```

걷는 경우에는 신발 사용 여부가 변하지 않는다.

```java
int nx = cur.x + dx[d];
int ny = cur.y + dy[d];

if (isInRange(nx, ny, n, m)
        && !trap[nx][ny]
        && !visited[nx][ny][cur.used]) {

    visited[nx][ny][cur.used] = true;
    q.offer(new Node(nx, ny, cur.time + 1, cur.used));
}
```

---

## 2️⃣ 신발로 두 칸 점프

```text
현재 위치 → 같은 방향으로 두 칸 뒤
```

이동 조건은 다음과 같다.

```text
1. 아직 신발을 사용하지 않았어야 한다.
2. 점프 도착 칸이 지도 범위 안이어야 한다.
3. 점프 도착 칸이 함정이면 안 된다.
4. 신발을 사용한 상태로 방문한 적이 없어야 한다.
```

중간 칸은 함정이어도 상관없다.

```text
현재 칸 → 함정 → 빈 칸
```

위와 같은 경우 신발 점프로 이동할 수 있다.

```java
if (cur.used == 0) {
    int jumpX = cur.x + dx[d] * 2;
    int jumpY = cur.y + dy[d] * 2;

    if (isInRange(jumpX, jumpY, n, m)
            && !trap[jumpX][jumpY]
            && !visited[jumpX][jumpY][1]) {

        visited[jumpX][jumpY][1] = true;
        q.offer(new Node(jumpX, jumpY, cur.time + 1, 1));
    }
}
```

## ⚠️ 주의할 점

### 1. `visited[x][y]`만 쓰면 안 된다

```java
boolean[][] visited;
```

이렇게 하면 같은 좌표를 한 번만 방문 처리하게 된다.

하지만 이 문제에서는 같은 좌표라도 다음 두 상태를 구분해야 한다.

```text
(x, y, 신발 안 씀)
(x, y, 신발 씀)
```

따라서 반드시 다음처럼 관리해야 한다.

```java
boolean[][][] visited;
```

---

### 2. 한 칸 앞이 함정이라고 바로 `continue`하면 안 된다

다음과 같은 코드는 위험하다.

```java
if (trap[nx][ny]) continue;
```

왜냐하면 한 칸 걷기는 불가능해도, 신발 점프는 가능할 수 있기 때문이다.

```text
현재 칸 → 함정 → 빈 칸
```

이 경우에는 걷기는 불가능하지만 신발 점프로는 이동할 수 있다.

따라서 **걷기 검사와 점프 검사는 독립적으로 처리**해야 한다.

---

### 3. BFS를 사용하는 이유

걷기와 신발 점프 모두 소요 시간이 `1`이다.

```text
한 칸 걷기: 1
두 칸 점프: 1
```

모든 이동 비용이 같으므로 BFS에서 가장 먼저 목적지에 도착하는 순간이 최소 시간이다.

따라서 이 문제는 다익스트라가 아니라 BFS로 충분하다.

---

## 🧩 핵심 Java 구조

### Node 상태

```java
static class Node {
    int x;
    int y;
    int time;
    int used;

    Node(int x, int y, int time, int used) {
        this.x = x;
        this.y = y;
        this.time = time;
        this.used = used;
    }
}
```

---

### BFS 시작 상태

```java
Queue<Node> q = new ArrayDeque<>();

q.offer(new Node(1, 1, 0, 0));
visited[1][1][0] = true;
```

시작점은 `(1, 1)`이다.

시작할 때는 아직 신발을 사용하지 않았으므로 `used = 0`이다.

---

### 목적지 도착 처리

```java
if (cur.x == n && cur.y == m) {
    return cur.time;
}
```

BFS이므로 목적지에 처음 도착한 시간이 최소 시간이다.

---

## 🧭 구현 흐름

```text
1. 함정 정보를 trap 배열에 저장한다.

2. visited[x][y][used] 배열을 만든다.

3. BFS 큐에 시작 상태를 넣는다.
   상태: (1, 1, 0초, 신발 안 씀)

4. 큐에서 현재 상태를 꺼낸다.

5. 현재 위치가 목적지라면 현재 시간을 반환한다.

6. 상하좌우 네 방향에 대해 다음 이동을 시도한다.
   6-1. 한 칸 걷기
   6-2. 신발을 아직 안 썼다면 두 칸 점프

7. BFS가 끝날 때까지 목적지에 도착하지 못하면 -1을 반환한다.
```

---

## ⏱️ 복잡도 분석

지도의 칸 수는 `n * m`이다.

각 칸마다 신발 사용 여부가 두 가지 존재한다.

```text
(x, y, 0)
(x, y, 1)
```

따라서 전체 상태 수는 최대 `2 * n * m`이다.

각 상태에서 상하좌우 네 방향을 확인한다.

$$
시간 \ 복잡도 = O(nm)
$$

$$
공간 \ 복잡도 = O(nm)
$$
