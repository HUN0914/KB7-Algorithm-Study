## 📝 문제 설명

총 $n$일 동안 외주 개발을 하여 최대한 많은 수익을 얻으려고 한다.

각 날짜마다 하나의 외주 작업이 주어지며, 해당 작업은 다음 정보를 가진다.

| 항목  | 의미                                                            |
| ----- | --------------------------------------------------------------- |
| $t_i$ | $i$번째 날에 시작할 수 있는 외주 작업을 완료하는 데 걸리는 기간 |
| $p_i$ | 해당 외주 작업을 완료했을 때 얻는 수익                          |

외주 작업은 다음 조건을 만족해야 한다.

- 하루에 시작할 수 있는 외주 작업은 해당 날짜의 작업 하나뿐이다.
- 여러 외주 작업의 기간이 서로 겹치면 안 된다.
- 작업 종료일이 $n$일을 넘어가면 해당 작업은 수행할 수 없다.
- 가능한 외주 작업들을 골랐을 때 얻을 수 있는 최대 수익을 구해야 한다.

---

```text
n
t1 p1

...
tn pn
```

| 항목  | 설명                              |
| ----- | --------------------------------- |
| $n$   | 전체 기간                         |
| $t_i$ | 외주 작업에 걸리는 기간           |
| $p_i$ | 외주 작업을 완료했을 때 얻는 수익 |

## 📤 출력

```text
가능한 외주 수익의 최대값
```

## 💡 핵심 아이디어

각 날짜마다 선택지는 두 가지이다.

| 선택                           | 설명                                      |
| ------------------------------ | ----------------------------------------- |
| 현재 날짜의 외주를 하지 않는다 | 현재까지의 최대 수익을 다음 날짜로 넘긴다 |
| 현재 날짜의 외주를 한다        | 작업 종료 날짜에 수익을 반영한다          |

즉, 날짜를 앞에서부터 보면서 현재 날짜까지의 최대 수익을 이용해 이후 날짜의 값을 갱신하면 된다.

이 풀이에서는 `time`, `profit` 배열을 따로 저장하지 않는다.

입력을 받는 즉시 현재 날짜의 외주 정보를 사용해 DP를 갱신한다.

---

## 🧠 DP 정의

$dp[i]$를 다음과 같이 정의한다.

```text
dp[i] = i번째 날까지 고려했을 때 얻을 수 있는 최대 수익
```

0-index 기준으로 보면 다음과 같다.

```text
day = 0  → 1일차
day = 1  → 2일차
...
day = n  → 모든 기간이 끝난 상태
```

따라서 정답은 `dp[n]`이다.

---

## 🔄 점화식

현재 날짜를 `day`라고 하자.

### 1. 현재 외주를 하지 않는 경우

현재 날짜의 일을 하지 않으면, 현재까지의 최대 수익을 다음 날짜로 넘긴다.

```text
dp[day + 1] = max(dp[day + 1], dp[day])
```

---

### 2. 현재 외주를 하는 경우

현재 외주 작업에 걸리는 시간이 `time`, 수익이 `profit`이라면 작업 종료 날짜는 다음과 같다.

```text
endDay = day + time
```

`endDay <= n`이면 기간 안에 끝낼 수 있으므로 선택 가능하다.

```text
dp[endDay] = max(dp[endDay], dp[day] + profit)
```

---

## 🧩 핵심 Java 코드

전체 제출 코드는 `Main.java`에 작성한다.

```java
int[] dp = new int[n + 1];

for (int day = 0; day < n; day++) {
    StringTokenizer st = new StringTokenizer(br.readLine());

    int time = Integer.parseInt(st.nextToken());
    int profit = Integer.parseInt(st.nextToken());

    // 현재 날짜의 외주를 하지 않는 경우
    dp[day + 1] = Math.max(dp[day + 1], dp[day]);

    int endDay = day + time;

    // 현재 날짜의 외주를 할 수 있는 경우
    if (endDay <= n) {
        dp[endDay] = Math.max(dp[endDay], dp[day] + profit);
    }
}

System.out.println(dp[n]);
```

## 🛠️ 구현 흐름

```text
1. n을 입력받는다.
2. dp 배열을 n + 1 크기로 만든다.
3. day = 0부터 n - 1까지 순회한다.
4. 현재 날짜의 time, profit을 입력받는다.
5. 현재 외주를 하지 않는 경우를 dp[day + 1]에 반영한다.
6. 현재 외주를 할 수 있다면 endDay에 최대 수익을 갱신한다.
7. 모든 날짜를 처리한 뒤 dp[n]을 출력한다.
```

## ⚠️ 주의할 점

### 1. `dp[day + 1]` 갱신이 필요하다

현재 날짜의 외주를 하지 않는 경우도 반드시 처리해야 한다.

```java
dp[day + 1] = Math.max(dp[day + 1], dp[day]);
```

이 코드가 있어야 이전 날짜까지 얻은 최대 수익이 다음 날짜로 전달된다.

---

### 2. 작업 종료 날짜가 $n$을 넘으면 안 된다

현재 작업의 종료 날짜는 다음과 같이 계산한다.

```text
endDay = day + time
```

이때 `endDay <= n`인 경우에만 해당 작업을 선택할 수 있다.

```java
if (endDay <= n) {
    dp[endDay] = Math.max(dp[endDay], dp[day] + profit);
}
```

---

### 3. 입력을 저장하지 않아도 된다

이 문제는 현재 날짜의 외주 정보만 있으면 바로 DP 갱신이 가능하다.

따라서 다음과 같은 배열은 필수가 아니다.

```java
int[] time = new int[n];
int[] profit = new int[n];
```

입력받는 즉시 처리하면 코드가 더 간결해진다.

---

### 4. 그리디로 풀면 안 된다

다음과 같은 기준으로 단순하게 선택하면 정답을 보장할 수 없다.

| 잘못된 기준                         | 이유                                    |
| ----------------------------------- | --------------------------------------- |
| 수익이 큰 작업부터 선택             | 기간이 길어 더 좋은 조합을 막을 수 있다 |
| 기간이 짧은 작업부터 선택           | 수익이 낮은 작업만 고르게 될 수 있다    |
| 수익 / 기간 비율이 큰 작업부터 선택 | 날짜 조합에 따라 손해가 될 수 있다      |

이 문제는 각 날짜의 선택 결과를 비교하며 최대값을 누적해야 한다.
